# 稍后细品：一个本地优先的网页稍后阅读工作台

我们每天都会遇到太多“值得之后再看”的内容。

问题不是不会收藏，而是收藏之后它们很快变成另一个被遗忘的角落。浏览器收藏夹太重，聊天窗口转发太散，传统稍后读工具又常常把“保存”做得很轻，把“回来整理”做得不够自然。

**稍后细品 LaterNest** 想解决的是这个小而真实的问题：把网页先放进一个安静的收件箱，再在合适的时间重新整理、回看、完成。

## 它是什么

LaterNest 是一个 Manifest V3 Chrome 插件。你可以在浏览网页时快速保存当前页面，并为它选择分类、备注和待看时间。

保存之后，所有内容会进入一个 Dashboard。你可以按时间线回看，也可以切到周视图，把这周积累的内容快速梳理出来。

## 设计原则

LaterNest 的核心原则是本地优先。

它没有账号系统，没有远程数据库，也不会默认上传你的保存内容。你的链接、备注、分类和设置都保存在自己的 Chrome 本地存储里。

如果你愿意，也可以配置自己的飞书或 Lark Webhook，让它把未看清单发送到你自己的群里。但这是可选能力，不是产品默认行为。

## 适合谁

- 每天会保存很多资料、文章、视频或工具页面的人。
- 希望把“稍后看”变成一个可整理工作流的人。
- 想要一个轻量、本地、可自己掌控数据的小工具的人。

## 主要功能

- 快速保存当前网页。
- 工作、学习、娱乐三类组织。
- 时间线视图，按日期回看。
- 周视图，整理一周内容。
- 搜索、筛选、收藏、完成、删除、延后。
- 批量操作，快速清理积压链接。
- 可选飞书或 Lark Webhook 同步。

## 隐私说明

LaterNest 默认不上传你的链接。

保存内容存储在本地 Chrome 中。只有当你主动配置 Webhook 并触发同步时，链接才会发送到你自己的飞书或 Lark 机器人。

这也是我把它做成插件，而不是一个需要登录的网站的原因：这个工具处理的是私人注意力，不应该默认进入别人的服务器。

## 安装方式

目前推荐从 GitHub 源码安装：

1. 下载仓库。
2. 打开 `chrome://extensions/`。
3. 打开开发者模式。
4. 选择“加载已解压的扩展程序”。
5. 选择 LaterNest 文件夹。

## 下一步

LaterNest 还会继续保持克制。

未来可能会加入更好的导出、更多整理视图和更细的提醒策略。但它不会为了复杂而复杂。它要做的事情仍然很简单：帮你把值得稍后细品的内容，真的带回到眼前。
